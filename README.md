# 运单箱号识别分拣工具源码

本项目用于从运单图片或 PDF 中离线识别集装箱箱号，并按识别结果分类输出文件和 Excel 汇总表。工具面向 Windows 本地离线使用场景，适合批量处理清关、物流、货代等业务中的运单箱号核对与文件分拣。

## 应用场景

- 批量识别清关运单、装箱资料、扫描件或图片中的集装箱箱号。
- 将文件自动分拣到 `正确识别`、`未识别`、`箱号错误` 等目录，减少人工逐个改名和分类。
- 对 OCR 结果生成 `识别结果.xlsx`，方便后续复核、汇总和留档。
- 对模糊件、箱号位置不固定的文件进行增强识别，并把不确定结果标记为待人工确认。
- 对已有箱号清单进行批量比对，快速发现缺失箱号、多余识别箱号或清单格式问题。

## 面向使用人员

- 清关/报关业务人员：用于核对一批运单文件是否覆盖应有箱号。
- 货代、物流、仓储操作人员：用于批量整理 PDF 或图片资料，并按箱号重命名输出。
- 单证、客服或运营人员：用于减少手工打开文件、查找箱号、重命名文件的重复工作。
- 内部质检/复核人员：通过 Excel 汇总表和“待确认”文件名快速定位需要人工复核的文件。
- 开发/运维人员：维护离线 OCR 依赖、打包版本和样本回归测试。

## 预期箱号清单会做什么

预期箱号清单是可选输入，用于“核对这一批文件是否识别齐全”，不是强制 OCR 答案，也不会单独覆盖程序识别结果。

选择预期箱号清单后，程序会：

- 在开始处理前读取清单，支持 `.txt`、`.csv`、`.xlsx`。
- 从清单内容中提取符合 ISO 6346 校验的箱号。
- 在界面中显示清单质量：有效箱号数量、重复箱号数量、无效格式数量。
- 如果清单没有任何可用箱号，会阻止开始处理并提示检查格式或校验位。
- 在单个文件输出前，如果结果是 `未识别` / `箱号错误`、当前没有唯一待确认箱号，程序会用 OCR 候选与清单做唯一匹配；若只命中一个清单箱号，则作为 `{箱号}-待确认` 文件名输出，但状态仍保持原状态。
- 文件处理完成后，把“正确识别”的箱号与清单做比对。
- 在日志中输出箱号比对结果：已匹配、缺失、多余识别、格式无效。
- 在 `识别结果.xlsx` 中额外生成“箱号比对”工作表，便于人工核对。

比对含义：

- 已匹配：清单里有，程序也正确识别到了。
- 缺失：清单里有，但本次结果没有正确识别到，可能需要查找未识别/箱号错误目录。
- 多余识别：程序正确识别到了，但清单里没有，可能是清单漏填、文件混入或识别结果需要复核。
- 格式无效：清单中无法提取出合法箱号的内容。

清单格式建议：

```text
GESU5903360
KSBU0042011
TEMU6779790
```

也可以使用 CSV 或 Excel；程序会扫描单元格内容并提取其中合法箱号。建议每行或每个单元格只放一个箱号，便于人工维护。

## 当前范围

- Python 项目骨架与 pytest 测试配置。
- ISO 6346 箱号校验、候选提取和非法箱号分类。
- 图片/PDF 输入扫描、Tesseract OCR、PDF 转图。
- 分类输出与 `识别结果.xlsx` 汇总。
- Tkinter 桌面界面。
- CLI 环境诊断、批量处理和样本基线验收。

## 本地验证

```powershell
. .\local-env.ps1
.\.venv\Scripts\python.exe -m pytest
.\.venv\Scripts\python.exe -m waybill_ocr diagnose
.\.venv\Scripts\python.exe -m waybill_ocr verify-samples
```

`verify-samples` 默认读取 `samples/input` 和 `samples/expected/baseline.csv`，输出到 `samples/actual`。

## v1.0.3 后续增强说明

- 样本回归：新增 `samples/cases` 分类目录，建议按 `clear`、`blurred`、`shifted`、`invalid-check-digit`、`no-container`、`multilingual-name` 存放脱敏样本；`verify-samples` 基线支持 `expected_status` 和 `allow_review_code`。
- 预期清单：选择清单后会展示有效箱号预览；处理完成后会生成 `缺失箱号清单.txt`，Excel 的“箱号比对”表按预期清单顺序展示“已识别 / 待确认命中 / 缺失”。
- GUI 体验：处理完成后可直接打开输出目录、识别结果 Excel、正确识别、未识别、箱号错误目录；软件会记住最近使用路径。
- 待确认整理：结果菜单支持查看待确认文件；双击使用系统默认程序打开，人工勾选后批量移动到正确识别目录；有预期清单时可自动整理唯一匹配且无冲突的候选。
- 错误说明：常见失败原因会转成更容易理解的说明，例如校验位错误、多个候选冲突、PDF 转图失败、OCR 组件失败或结果表被占用。
