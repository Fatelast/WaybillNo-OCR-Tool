# 版本说明

## v1.0.2

当前版本已具备本地离线可用的 GUI 批量识别流程：

- 支持图片和 PDF 输入，PDF 会先转图再 OCR，不依赖 PDF 内嵌文本层。
- 支持最多 2 组输入文件夹并行处理，每组独立生成输出分类和 `识别结果.xlsx`。
- 输出文件夹可不选，未选时自动使用 `输入文件夹名_识别输出`。
- GUI 会阻止两组任务使用同一个输出目录，避免并行写入同一份结果表。
- 识别结果新增“相对路径”列，历史结果跳过优先按相对路径匹配，避免子目录同名文件被误判为已处理。
- 速度模式在 GUI 中展示取舍说明：稳定模式准确率优先，均衡模式默认推荐，快速模式速度最快但可能漏识别位置特殊的箱号。
- 疑似候选和猜测型修正仅写入备注，最终结果仍只接受严格 ISO 6346 校验通过的箱号。
- 识别失败文件会跳过并继续处理后续文件；区域裁剪异常会写入未识别备注便于排查。
- 结果表被 Excel/WPS 占用时，自动改写本轮备用 Excel，不中断 OCR 处理。
- 支持预期箱号清单前置校验和箱号比对工作表。

## 打包状态

- 使用 `app.spec` 构建 Windows GUI 本地 `dist` 目录，产物名称为 `运单箱号识别分拣`。
- 打包会随包带入 `tools`、`resources` 和 `docs`。
- 本地 `dist` 仅用于本机试用；正式离线交付包需要另行构建和压缩。

## 已知限制

- 快速模式不执行完整网格兜底，可能漏识别位置特殊或图像质量较差的箱号。
- 目前不自动把 O/0、I/1、B/8 等猜测替换作为最终箱号，只作疑似信息留给人工复核。
- 真实样本准确率仍需要持续通过样本回归库验证和迭代。
