# 版本说明

## v1.0.2

当前版本已具备本地离线可用的 GUI 批量识别流程：

- 支持图片和 PDF 输入，PDF 会先转图再 OCR，不依赖 PDF 内嵌文本层。
- 支持最多 2 组输入文件夹并行处理，每组独立生成输出分类和 `识别结果.xlsx`。
- 输出文件夹可不选，未选时自动使用 `输入文件夹名_识别输出`。
- GUI 会阻止两组任务使用同一个输出目录，避免并行写入同一份结果表。
- 识别结果主表精简为原始文件名、识别箱号、识别状态、处理耗时和备注；相对路径、失败原因、识别来源改写入隐藏内部索引，历史跳过仍优先按相对路径匹配。
- 速度模式在 GUI 中展示取舍说明：稳定模式准确率优先，均衡模式默认推荐，快速模式速度最快但可能漏识别位置特殊的箱号。
- 疑似候选和猜测型修正仅写入备注，最终结果仍只接受严格 ISO 6346 校验通过的箱号。
- 识别失败文件会跳过并继续处理后续文件；区域裁剪异常会写入未识别备注便于排查。
- 未识别或箱号错误文件如存在唯一安全复核候选，会输出为 `候选箱号-待确认`，方便人工验收。
- 结果表被 Excel/WPS 占用时，自动改写本轮备用 Excel，不中断 OCR 处理。
- 支持预期箱号清单前置校验和箱号比对工作表。

## 打包状态

- 使用 `app.spec` 构建 Windows GUI 本地 `dist` 目录，产物名称为 `运单箱号识别分拣`。
- 打包会随包带入 `tools`、`resources` 和 `docs`。
- 本地 `dist` 仅用于本机试用；正式离线交付包需要另行构建和压缩。

## 已知限制

- 快速模式不执行完整网格兜底，可能漏识别位置特殊或图像质量较差的箱号。
- 目前不自动把 O/0、I/1、B/8 等猜测替换作为最终箱号，只作疑似信息留给人工复核。
- 真实样本准确率仍需要持续通过样本回归库验证和迭代。

## 补充优化

- 增加 scripts/cleanup_workspace.ps1，用于预览并清理测试、打包和 Python 缓存生成物。
- 多个单数字校验修正候选时不自动采用，改为在备注中提示候选，降低误改名风险。

## v1.0.3 后续增强说明

- 处理性能：只配置一组任务时，均衡/快速模式最多并行识别 2 个文件；两组任务仍各处理 1 个文件。分类复制保持逐文件执行，Excel 改为每 5 个新结果分批写入，并在结束或取消时强制保存；写入失败不会被误判为已保存，任务结束前会重试。
- 增强识别：均衡模式处理 PDF 时，合法候选需同时获得 400 DPI 与基础分辨率证据后才提前结束；跨分辨率冲突、证据不足及稳定模式继续执行完整增强兜底。
- 样本回归：支持本地私有 `baseline.local.csv`，当前已用 3 个真实业务样本建立初始基线并通过验收；样本文件不会进入 Git。
- 输出幂等：同一输入文件重新处理时复用或替换上次输出路径；识别状态变化时会移除旧分类副本，避免累计 `-1`、`-2` 文件。
- 异常恢复：逐文件结果写入本机恢复日志，Excel 使用临时文件原子替换；异常退出后重新启动可恢复尚未写入主结果表的记录。
- 预期清单：选择清单后会展示有效箱号预览；处理完成后会生成 `缺失箱号清单.txt`，Excel 的“箱号比对”表按预期清单顺序展示“已识别 / 待确认命中 / 缺失”。
- GUI 体验：处理完成后可直接打开输出目录、识别结果 Excel、正确识别、未识别、箱号错误目录；启动时表单保持为空，浏览文件时自动定位到最近使用目录。
- 待确认文件整理：支持双击查看待确认文件、人工勾选后批量整理；有预期清单时支持唯一匹配候选自动整理，冲突文件保持原状。
- 错误说明：常见失败原因会转成更容易理解的说明，例如校验位错误、多个候选冲突、PDF 转图失败、OCR 组件失败或结果表被占用。
