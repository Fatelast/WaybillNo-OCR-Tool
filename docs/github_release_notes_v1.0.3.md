# 运单箱号识别分拣 v1.0.3

本版本提供 Windows 本地离线版程序，用于批量识别运单 PDF 和图片中的集装箱箱号，自动完成文件重命名、结果分类和 Excel 汇总。适用于清关、报关、货代、物流和仓储等批量单证处理场景。

## 下载与启动

请下载本次 Release 附件：

- `运单箱号识别分拣_v1.0.3_离线版.zip`

解压后双击 `运单箱号识别分拣.exe` 启动。程序已包含 Tesseract OCR 和 Poppler，无需安装 Python 或其他开发环境。

不要把 EXE 单独移出解压目录；如需移动程序，请移动整个文件夹，并保持 EXE 与 `_internal` 目录在一起。

## 主要更新

### 识别质量

- 支持 `.pdf`、`.jpg`、`.jpeg`、`.png`、`.bmp`、`.tif`、`.tiff` 文件。
- 按整页、优先区域和网格区域分阶段识别，兼顾常见版式与箱号位置不固定的文件。
- 均衡和稳定模式会对模糊件、高风险合法候选、无效校验位候选执行增强 OCR 和交叉验证。
- 候选判断综合区域、上下文、重复证据和 ISO 6346 校验，降低“错误但合法箱号”被直接归为正确识别的风险。
- 识别到重复箱号时，相关文件会转入 `箱号错误` 等待复核，避免多个文件被错误归到同一箱号。
- 不会把 `O/0`、`I/1`、`B/8`、`G/6`、`T/7` 等猜测型替换直接作为最终正确结果。

### 处理效率

- 支持最多 2 组“输入文件夹 -> 输出文件夹”并行处理，两组任务输出相互隔离。
- 提供稳定、均衡、快速三种速度模式，并在界面中显示简短的适用场景。
- 均衡模式会在获得足够候选证据后提前结束低价值区域识别，减少失败文件的无效 OCR 调用。
- 优化 Excel 增量写入频率，并使用恢复记录保护尚未写入结果表的数据。
- 支持停止当前任务；停止时会中断正在执行的 OCR 子进程，不必等待整批文件完成。

### 失败件与待确认整理

- 单个文件处理失败时会跳过该文件并继续后续任务，不影响另一组并行任务。
- `未识别`、`箱号错误`文件存在唯一安全候选时，会命名为 `{箱号}-待确认`，便于操作员直接验收。
- “打开结果”菜单支持重新识别失败文件，无需重新处理已成功文件。
- 新增待确认文件列表，可双击使用系统默认程序查看文件，勾选后批量移动到 `正确识别` 并去除 `-待确认`。
- 待确认列表扩大勾选区域，提供选中行高亮、全选、取消全选、实时已选数量和醒目的操作提示。
- 上传预期箱号清单后，可自动整理唯一匹配、校验合法且不存在重复冲突的待确认文件。

### 预期箱号清单

- 清单为可选项，支持 `.txt`、`.csv`、`.xlsx`。
- 选择清单后会显示有效箱号、重复项、无效项和部分箱号预览。
- 失败结果没有唯一待确认候选时，可用 OCR 候选与预期清单进行唯一匹配；命中后仍保持原失败状态，仅用于 `-待确认` 命名。
- 输出 `缺失箱号清单.txt`，并在 Excel 的“箱号比对”工作表中按清单顺序展示“已识别 / 待确认命中 / 缺失”。
- 清单不会凭空指定 OCR 结果，也不会把待确认文件自动标记为正确识别。

### GUI 与可靠性

- 输出文件夹可不选；程序会在输入文件夹同级创建 `{输入文件夹名}_识别输出`。
- 每组任务显示处理进度、成功/未识别/箱号错误数量和执行时长。
- 处理完成后可直接打开输出目录、结果 Excel 和各分类目录。
- 启动时任务表单保持为空；点击“浏览”时会自动定位到最近使用的输入目录、输出目录或清单目录。
- 再次处理同一目录时可复用历史结果并跳过已成功文件；同一输入文件重新处理时会替换旧输出，避免生成大量重复副本。
- 异常退出后可恢复尚未写入 Excel 的处理记录。
- `识别结果.xlsx` 被 Excel/WPS 占用时，自动改写本轮备用结果表，不中断 OCR。
- Excel 主表精简为业务验收字段，错误行使用红底标记，备注会截断过长候选内容并显示易懂的失败说明。
- 不再生成识别证据截图和证据文件夹，减少输出体积和磁盘占用。

## 速度模式

- **稳定模式**：适合模糊、版式复杂或准确率优先的文件，识别更充分，耗时较长。
- **均衡模式**：适合日常批量处理，在速度和识别覆盖之间取得平衡，默认推荐。
- **快速模式**：适合清晰、版式较统一的文件快速粗筛，位置特殊或模糊箱号可能漏识别。

## 使用说明

1. 解压 `运单箱号识别分拣_v1.0.3_离线版.zip`。
2. 双击 `运单箱号识别分拣.exe`。
3. 选择任务 1 的输入文件夹；任务 2 可选。
4. 输出文件夹可不选，不选时程序会自动生成。
5. 按需选择预期箱号清单和速度模式。
6. 点击“开始处理”。
7. 处理完成后，通过“打开结果”查看 `正确识别`、`未识别`、`箱号错误`、`识别结果.xlsx`。
8. 对 `-待确认` 文件可进入“待确认文件”逐个查看并批量整理；失败文件可选择“失败文件重新识别”。

## 已知限制

- OCR 准确率仍受扫描清晰度、压缩失真、印章遮挡、字体粘连、页面倾斜和箱号位置影响。
- 快速模式不会执行完整增强兜底，模糊文件建议改用均衡或稳定模式重新识别。
- `-待确认` 表示程序提供了复核候选，不代表箱号已经人工确认。
- 没有预期清单时，程序不会自动把待确认文件整理为正确识别，必须由操作员查看并勾选确认。
- 输出结果表被占用时，程序会创建备用 Excel；关闭原文件后可在下一批任务中恢复使用主结果表。
