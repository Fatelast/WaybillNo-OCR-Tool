# 项目知识快照

## 架构链路

项目核心链路为：文件扫描 -> 图片/PDF 读取 -> OCR -> 箱号候选提取 -> 候选评分 -> ISO 6346 校验 -> 分类输出 -> Excel 汇总。样本验收链路在批处理之后追加 baseline 对比，不改变识别主流程。

## 当前模块

- `container_code.validator`：负责 ISO 6346 校验位计算和合法性判断；当前业务只接受 `3 位大写字母 + U + 7 位数字` 的集装箱箱号。
- `container_code.extractor`：负责从 OCR 文本或文件名中提取 `XXXU1234567` 形态的箱号候选，也能提取格式完整但校验失败的候选和疑似修正候选。
- `container_code.candidate_selector`：负责对 OCR 候选打分，优先选择靠近箱型代码、箱号关键词或局部区域 OCR 的候选。
- `file_scanner`：负责递归扫描输入目录中的图片和 PDF。
- `image_loader`：负责把图片路径或 PDF 转图后的临时图片路径提供给 OCR。
- `image_regions`：负责整页、优先区域、网格区域和增强区域生成，用于降低整页 OCR 噪声影响。
- `ocr.base`：定义 OCR 引擎协议和识别结果结构。
- `ocr.tesseract_engine`：封装 Tesseract 调用，支持取消令牌、`psm` 覆盖、字符白名单和重试策略。
- `pipeline`：编排单文件识别流程；先收集基础候选，再按速度模式决定网格兜底、增强 OCR、待确认候选二次验证和多候选评分；成功结果只接受严格校验通过且证据足够的箱号。
- `batch_processor`：负责批量扫描、逐文件处理、分类复制、Excel 增量写入、取消处理和 Excel 占用备用表兜底。
- `sample_verifier`：负责读取 `samples/expected/baseline.csv`，复用批处理结果并对比样本期望，用于端到端验收。
- `output.classifier`：负责按识别状态复制文件到分类目录；正确识别文件按 `container_code + 原扩展名` 命名，未识别/箱号错误如有 `review_code` 则按 `复核候选-待确认 + 原扩展名` 命名，否则保留原名。
- `review_confirmation`：扫描输出目录中的待确认文件；支持人工勾选后移动并去掉 `-待确认`，或基于预期清单唯一匹配自动整理，不覆盖已有正确识别文件。
- `output.excel_writer`：负责生成 `识别结果.xlsx`；主表只展示原始文件名、识别箱号、识别状态、处理耗时和备注，隐藏 `内部索引` 保存相对路径、来源、失败原因等程序诊断字段。
- `ui.main_window`：负责 Tkinter 目录选择、两组任务配置、速度模式、停止处理、进度文本和日志展示，不直接实现识别规则。
- `models`：集中定义跨模块传递的数据结构。
- `config.default_config`：自动解析运行根目录、随包 `tools/tesseract/tesseract.exe`、`tools/poppler`，并支持环境变量覆盖。
- `diagnostics`：负责检查 Python 依赖、Tesseract、Poppler 是否可用，并生成 UI 可展示的诊断消息。
- `cli`：提供 `diagnose`、`batch` 和 `verify-samples` 命令；无参数时仍由 `app.run` 启动 Tkinter GUI。
- `app.spec`：PyInstaller Windows 打包配置，选择性携带 Tesseract、Poppler、`resources` 与文档资源，减少离线包体积。
- `scripts/cleanup_workspace.ps1`：用于预览或清理 `.tmp`、`.pytest-*`、`pytest-cache-files-*`、`build`、`__pycache__` 等生成物；默认不删除 `dist`，需要时通过 `-IncludeDist` 显式包含。
- `docs/操作说明.md` 与 `docs/版本说明.md`：面向交付包的基础用户说明和版本状态说明。

## 注意事项

- `HNKU6331795` 是当前核心验收样例。
- `KSBU0042011`、`GESU5903360` 是用户实测 PDF 截图中的合法箱号样例，输出正确识别文件时应按箱号命名。
- `HINKU6331795` 因结构不符合 `4 位大写字母 + 7 位数字`，不应被识别为合法箱号。
- `TEMU6779790`、`MEDU4185918` 属于模糊样本增强识别重点；增强结果与基础候选冲突时需要避免把错误但合法的候选直接归为正确识别。
- `YYCU6003610` 这类从校验失败候选推导出的箱号只能在唯一安全候选时作为待确认命名；存在多个单数字修正候选时不自动采用，只写入备注供人工复核。
- OCR 与 PDF 相关第三方依赖使用延迟导入，避免未安装 Tesseract/Poppler 时影响核心规则测试。
- 配置覆盖环境变量：`WAYBILL_OCR_TESSERACT_CMD`、`WAYBILL_OCR_POPPLER_PATH`、`WAYBILL_OCR_RETRIES`。
- 缺少 `pytesseract` 或 `pdf2image` 时会抛出中文 `RuntimeError`，用于 UI 展示可读失败原因。
- Tkinter UI 后台线程只调用批量编排层，界面更新通过 `root.after` 回到主线程。
- 后续 OCR 优化必须以样本集和基线数据为依据。
- `samples/expected/baseline.csv` 使用 `filename,expected_code,should_recognize,quality_tag,notes` 字段；`should_recognize=false` 表示该样本不应识别成功。
- `python -m waybill_ocr verify-samples` 默认读取 `samples/input`、写入 `samples/actual`，并按 baseline 返回验收结果；失败时退出码为 `1`。
- UI 开始处理时会先输出环境诊断；缺失项不直接阻断流程，但后续 OCR/PDF 转图会给出明确失败原因。
- `python -m waybill_ocr diagnose` 在缺少依赖或离线工具时返回退出码 `1`，用于打包前自动验收。
- PDF 转图按页懒加载，每次只转换当前页，以便识别成功后提前停止后续页处理。
- 分类复制会在同名冲突时追加 `-1`、`-2` 后缀，避免覆盖已有输出文件。
- 批量处理会排除位于输入目录内部的输出目录，避免重复扫描上次输出。
- 校验失败但格式完整的 `XXXU1234567` 箱号会归为 `箱号错误`，失败原因为 `INVALID_CHECK_DIGIT`；如果存在唯一安全复核候选，输出文件名会带 `-待确认`。
- 速度模式分层：快速模式只做清晰文件粗筛并保留待确认候选；均衡模式对失败件做有限复核；稳定模式对模糊件执行更多增强 OCR 和多 PSM 验证。
- 清理脚本默认是预览模式，只有添加 `-Apply` 才会删除生成物；删除项目文件仍需按对话确认后执行。
