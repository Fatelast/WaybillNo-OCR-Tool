# 操作说明

## 使用前准备

1. 确认程序目录中存在 `tools/tesseract/tesseract.exe`。
2. 确认程序目录中存在 Poppler 的 `pdftoppm.exe`，可放在 `tools/poppler`、`tools/poppler/bin` 或 `tools/poppler/Library/bin`。
3. 准备一个输入文件夹，放入需要识别的图片或 PDF。
4. 输出文件夹可不准备；程序默认在输入文件夹同级创建 `输入文件夹名_识别输出`。

## 操作步骤

1. 启动 `运单箱号识别分拣.exe`。
2. 点击“浏览”选择任务 1 输入文件夹，界面会立即显示可处理文件数量并自动补全输出目录；只有检测到支持文件且路径校验通过后，“开始处理”才会启用。
3. 如需自定义输出目录，再点击输出文件夹右侧“浏览”。
4. 需要两组并行处理时点击“添加第二组任务”；不需要时保持折叠即可。
5. 预期箱号清单可直接选择，仅用于核对缺失，不会覆盖识别结果；速度模式和样本验收位于开始处理按钮附近。
6. 点击“开始处理”，等待日志显示处理完成。
7. 使用完成摘要旁的快捷按钮查看待确认文件、正确识别或输出目录；Excel 等其他结果位于“打开结果”菜单。

已有结果的输出目录再次启动时，程序会提供三个选择：
- “继续未完成文件（推荐）”：跳过已有成功结果。
- “重新识别当前输入内全部文件”：重新处理当前输入中的文件，不删除输入原件。
- “取消”：不启动本次任务。两组任务统一采用同一次选择。

## 环境诊断

点击“开始处理”后，日志会先输出运行环境诊断：

- `[OK]` 表示依赖或工具可用。
- `[缺失]` 表示需要补齐对应 Python 依赖或离线工具。

如果存在缺失项，后续 OCR 或 PDF 转图可能失败，请优先按日志提示补齐。

## 速度模式

- 快速模式：适合清晰文件快速粗筛；不做深度复核，失败件更多保留为待确认或未识别。
- 均衡模式：默认推荐；对失败件做有限复核，兼顾速度和成功率。
- 稳定模式：适合模糊文件；会进行更多增强识别，速度更慢，但更容易识别出低清晰度箱号。
- 只配置一组任务时，均衡/快速模式会同时识别最多 2 个文件；配置两组任务时每组各处理 1 个文件，总并发仍为 2。
- 均衡模式仅在多个独立区域反复得到同一合法箱号且没有冲突时提前结束增强识别；证据不足时仍执行完整兜底。
## 识别规则

- 当前识别目标为集装箱箱号，格式为 `3 位大写字母 + U + 7 位数字`，并通过 ISO 6346 校验位校验。
- PDF 会先转换为图片再 OCR；程序不依赖 PDF 内嵌文本层，也不要求固定版式。
- 当整页 OCR 没有强候选时，会进一步识别重叠网格区域，并优先选择靠近箱型代码（例如 `45G1`、`22G1`）或箱号上下文的候选。

## 输出内容

输出文件夹中会生成：

- `正确识别`：识别到合法箱号的文件，文件名会改为 `箱号 + 原扩展名`，例如 `KSBU0042011.pdf`。
- `未识别`：没有识别到合法箱号或处理失败的文件；如程序能给出唯一待复核候选，文件名会改为 `候选箱号-待确认 + 原扩展名`，否则保留原始文件名。
- `箱号错误`：识别到格式完整但 ISO 6346 校验失败的箱号文件；如程序能给出唯一待复核候选，文件名会改为 `候选箱号-待确认 + 原扩展名`，否则保留原始文件名。
- `识别结果.xlsx`：识别汇总表，主表展示原始文件名、识别箱号、识别状态、处理耗时和备注；诊断字段写入隐藏内部索引，不影响人工验收。 分类文件会逐个输出，结果表每累计 5 个新结果更新一次，任务结束或取消时会强制保存剩余结果。
- 待确认整理：处理完成后点击“查看待确认”或结果菜单中的“待确认文件”。双击文件可用系统默认程序查看，勾选人工确认的文件后批量移动到 `正确识别`，并去掉 `-待确认`。上传预期清单时，可使用“按预期清单自动整理”处理唯一匹配且无冲突的候选。
- 上传预期清单且存在唯一、合法、无冲突候选时，完成区会显示“安全整理（N）”；结果入口只在处理完成后显示。

如果多个文件识别到同一个箱号，后续文件会自动追加 `-1`、`-2` 后缀，避免覆盖已有输出。

## 清理临时文件

项目提供安全清理脚本，默认只预览将要清理的内容，不会删除文件：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\cleanup_workspace.ps1
```

确认列表无误后，再执行实际清理：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\cleanup_workspace.ps1 -Apply
```

默认会清理 `.tmp`、`.pytest-*`、`.pytest_cache`、`pytest-cache-files-*`、`build`、`__pycache__` 等生成物；不会清理 `.venv` 和 `dist`。如需同时清理本地 `dist` 构建产物，可额外添加 `-IncludeDist`。
## 常见问题

- 如果提示缺少 `pytesseract` 或 `pdf2image`，请先安装 `requirements.txt` 中的依赖。
- 如果 PDF 无法转换，请检查 Poppler 是否已放入 `tools/poppler`。
- 如果图片无法 OCR，请检查 Tesseract 是否已放入 `tools/tesseract`。
- 如果处理速度变慢，通常是因为整页 OCR 没有强候选，程序进入了网格区域识别。

## 命令行用法

开发或验收时可以使用命令行入口：

```powershell
$env:PYTHONPATH='src'
python -m waybill_ocr diagnose
```

该命令会输出环境诊断；如果存在缺失项，退出码为 `1`。

批量处理命令：

```powershell
$env:PYTHONPATH='src'
python -m waybill_ocr batch --input D:\input --output D:\output
```

样本基线验收命令：

```powershell
$env:PYTHONPATH='src'
python -m waybill_ocr verify-samples
```

默认读取 `samples/cases`、写入 `samples/actual`。程序优先读取仅存于本机的 `samples/expected/baseline.local.csv`，不存在时回退到 `baseline.csv`。验收失败时会输出具体样本差异并返回退出码 `1`。

无命令行参数时，`python -m waybill_ocr` 会启动桌面界面。

## v1.0.3 后续增强说明

- 样本回归：新增 `samples/cases` 分类目录，建议按 `clear`、`blurred`、`shifted`、`invalid-check-digit`、`no-container`、`multilingual-name` 存放脱敏样本；`verify-samples` 基线支持 `expected_status` 和 `allow_review_code`。
- 预期清单：选择清单后会展示有效箱号预览；处理完成后会生成 `缺失箱号清单.txt`，Excel 的“箱号比对”表按预期清单顺序展示“已识别 / 待确认命中 / 缺失”。
- GUI 体验：处理完成后可直接打开输出目录、识别结果 Excel、正确识别、未识别、箱号错误目录；启动时表单保持为空，浏览文件时自动定位到最近使用目录。
- 错误说明：常见失败原因会转成更容易理解的说明，例如校验位错误、多个候选冲突、PDF 转图失败、OCR 组件失败或结果表被占用。
